## Аргументы программы
Для обработки аргументов используется класс ProcessProgramArguments. Принимает аргументы, при необходимости проверяет допустимые значения. Этот класс использует класс ProgramArguments для хранения значений аргументов по умолчанию и изменённых (всех необходимых параметров программы). Детальные пояснения не требуются, так как работа этих классов очевидна и код прокомментирован достаточно.

Используются следующие аргументы (в скобках указаны значения по умолчанию):

/c &emsp; - &emsp; Путь файла конфигурации (config.json)\
/r &emsp; - &emsp; Путь файла запросов (requests.json)\
/a &emsp; - &emsp; Путь файла ответов (answers.json)\
/dp &emsp; - &emsp; Признак формирования базы документов или путей файлов документов (yes)\
/l &emsp; - &emsp; Путь файла конфигурации логирования (logger.json)\
/m &emsp; - &emsp; Путь файла конфигурации очереди сообщений (messageQueue.json)\
/lc &emsp; - &emsp; Логировать события в консоль (yes)\
/p &emsp; - &emsp; Количество знаков после запятой (6)\
/f &emsp; - &emsp; Ширина вывода (2)\
/t &emsp; - &emsp; Желаемое количество потоков (количество ядер минус два)\
/e &emsp; - &emsp; Максимальное количество непрочитанных файлов (0)\
/? &emsp; - &emsp; Справка
\
\
Использование очевидно и не требует пояснений.
